# Claude-No-Compact
Solution for preventing Claude from automatically compressing context.

### 方案1 —— 交接简报

你是一位即将轮岗的资深项目资助经理。现在，你需要为你即将接替你工作的"新同事"(另一个AI对话)准备一份完整、清晰、无遗漏的工作交接简报。

这份简报的目的是让你的新同事能在最短的时间内，全面、准确地理解我们整个对话的背景、目标、过程、决策和当前状态，确保他能无缝衔接工作，避免重复提问和讨论。

请严格按照以下【七个部分】的Markdown格式生成这份简报，每个部分都必须经过深思熟虑，确保内容详实、重点突出。

工作交接简报
1. 核心目标与项目概述  （用一两句话总结我们这次对话最终想要达成的核心目标是什么？我们正在做什么项目或解决什么问题？）
2. 关键决策与最终共识  （以列表形式，清晰地列出我们已经讨论并确定的、不可更改的结论。这是最重要的部分，必须准确无误。）*决策1：...*  *决策2：...*
3. 已解决的核心问题与方案  （列出我们在对话中遇到的主要挑战或问题，以及我们最终是如何决定解决它们的。）*问题：...* 解决方案...
4. 主要脑暴路径与探索思路  （简述我们讨论过哪些主要的想法、方向或思路，包括那些【最终被放弃的】。请简要说明放弃它们的原因，这能避免新同事重复提出无效建议）
5. 待办事项与下一步行动  （明确指出新同事需要着手的下一项具体任务是什么？我们对话中断在了哪个步骤？）
6 .用户画像与协作偏好  （根据我们的互动，描述一下“我”用户的特点。例如：我的沟通风格是怎样的？我更看重效率还是创造力？我对哪些方面有明确的偏好或要求？这能帮助新同事更好地与我协作。）
7. 关键信息与片段  （如果对话中出现了任何必须完整保留的【关键数据、代码、引文或特别指令】，请将它们原封不动地复制到这里）

### 方案2 —— 手动管理上下文

不要依赖自动压缩（compact），要手动 /clear + 重载重要信息

方法：
让 Claude 把当前状态写成一个 .md 文件，这样内容清晰可见，也不会被过度总结。当压缩次数超过两次，使用/clear 清空上下文后，再让 Claude 重新读取这个 md 文件继续工作

### 方案3 —— 创建“人物角色（persona）”来更容易察觉上下文腐烂

一开始给 Claude 一个带戏剧性的个性（例如模仿某位著名工程师）。当 Claude 不再保持这个风格 → 说明上下文已经腐烂，需要重置。

### 方案4 —— 任务拆分成更小步骤

把大工作流程拆成小任务，每个任务执行完就清理上下文，清理完成后再继续下一步。这样能够避免堆积大量历史导致腐烂，这也是为什么我推荐spec规范开发的原因。
